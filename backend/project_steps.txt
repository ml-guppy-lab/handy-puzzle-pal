Step 2: Draw dynamic rectangle between hands + size check 

* Get the leftmost and rightmost points from both hands (use wrist or index base).
* Draw a rectangle using cv2.rectangle.
* Calculate area: rect_area = width * height
* Screen area = frame.shape[0] * frame.shape[1]
* When rect_area >= 0.4 * screen_area → trigger snapshot (save the cropped region inside the box as original_img).
* Add text overlay: “📸 SNAPSHOT TAKEN!” for 2 seconds.

Step 4: Detect pinch gesture (index + thumb close) (2 hours)

* Calculate distance between thumb tip (landmark 4) and index tip (landmark 8).
* If distance < 30 pixels → “pinch detected”.
* Add a small circle at the pinch point so you can see it.
----Reel: “Pinch = click. No mouse needed!”

Step 5: Build drag-and-drop puzzle logic (3–4 hours – the fun part)

* Keep the 3x3 grid as a list of tiles + their current positions.
* When pinch is detected:
    - Map hand position (use index finger tip) to the nearest grid cell.
    - “Pick up” that tile (store it in a selected_tile variable).
    - While pinched, move the tile with your hand (draw it at current hand position).
    - When pinch released → drop into the nearest empty cell (swap logic).

* Draw the entire puzzle every frame with cv2 rectangles + cv2.putText for numbers 1-9 (for debugging).
* Learning: State machine (idle → selecting → dragging → dropping) + coordinate mapping.
Pro tip: Start with mouse first (for testing), then replace mouse coords with hand coords.
